- 結論：**端到端管線可運作、測試全綠，但「肉眼品質 Gate」未過**；依使用者決定**暫停**，

## 1. 目標與當初假設

把 FB 3D Photo 小角度視差下的 **disocclusion 空洞**根治：沿 depth 斷崖切 2–3 層，
背景層用既有 C1（DepthAwareInpainter）**預先補洞**，前端在前景滑開時露出預填背景。
假設：這條（純 CPU、複用既有接口、零重依賴）能在低配硬體上做出「明顯優於單層視差」的縱深。

## 2. 實際做了什麼（皆已 commit、可運作）

- 後端 `src/core/ldi.py`：`LDIBuilder`（depth 分位數斷崖分層 + 背景層複用 C1 預填）+ Provider 單例。
- 後端 `POST /ldi`：回原圖 rgb/depth + 預填背景 bg + 完整 layers。
- 前端獨立「LDI 模式」（與視差/mesh 並存）。
- 測試 100 passed（12 單元 + 5 整合 + 83 既有）；`npm run build` 綠。

## 3. 三輪實機驗證與每輪症狀

| 輪次 | 作法 | 實機症狀 | 根因 |
|---|---|---|---|
| 1 | 多層 shader：sampler 陣列 + 動態迴圈 | **檢視視窗整片空白** | GLSL ES 對「sampler 陣列變數索引 / 動態迴圈上限」相容性差 → shader 編譯失敗 → three.js 靜默渲染空白（誤判為「切換未刷新」） |
| 2 | 改展開具名 uniform + **每層剛性平移** | 無洞無錯色，但**前景被整塊「拔出」成紙板、與背景脫離**；拖曳方向/手感不對 | 把離散層當剛體整塊平移 = 經典 cardboard cutout；且用「層深度中點」當平移量導致「背景比前景動更多」的反向錯覺 |
| 3 | 改回**連續逐像素 depth 位移**（= 視差模式數學）+ 破洞處取預填背景 bg | 紙板感消失、方向手感正確，但**脫影/拖影仍在，且整體與視差模式差異不大** | 見下節（本質問題） |

## 4. 為什麼第三輪「與視差模式差異不大、且仍有脫影」（核心檢討）

1. **小角度下 disocclusion 只是邊緣細條**：LDI 與視差模式**共用同一條連續位移路徑與同一張
   原圖**，只有在「取樣落到被掀開的破洞」那一小撮邊緣像素才改取預填背景。小角度時這區域
   僅佔極少數像素 → 整體觀感自然≈視差模式。

2. **脫影來自「資料」而非 shader**：預填背景 `bg` 是 C1 的**古典擴散補繪**，對「整張床/花瓶」
   這種**超大洞**會糊成放射狀條紋（早期背景層輸出圖已可見「X」狀 smear）。所以破洞分支即使
   正確觸發，取到的也是糊的內容 → 你看到的脫影。**shader 忠實呈現了一張低品質的填補圖**。

3. **架構天花板（最關鍵）**：「單張 depth map 位移 + 補洞」這一族方法（FB 3D Photo、單圖 LDI）
   **本質上只能做小幅變形**——學界明言被遮擋內容無法真正還原，故位移刻意保守、邊緣 artefact
   是內生的。要真正拉開與視差模式的差距，需要**換掉其中一塊資料/表示**，而非調 shader：
   - 把 `bg` 的補繪器換成**真正的生成式 inpainting**（LaMa / diffusion），讓大洞填得可信；
   - 或換成**真 3D 表示**（3DGS），但已於前一 spike 因 8GB Blackwell + 授權 + 生態而結論 no-go。

## 5. 哪些是對的、可保留（重啟時不必重做）

- **管線骨架對**：`AbstractLDIBuilder` + Provider + `/ldi` + 獨立前端模式，接口乾淨、零破壞既有路徑。
- **連續位移渲染對**：第三輪的 shader 是正確方向（與業界一致），不要再回去做剛性分層平移。
- **「破洞才取預填背景」的分支對**：邏輯正確，瓶頸在 `bg` 的**填補品質**，不在這段邏輯。
- 結論：**唯一真正要換的零件 = 背景補繪品質**。LDIBuilder 已用 Provider 化的 inpainter 注入，
  換 LaMa/diffusion 是「替換 builder 的 inpainter」而非重寫管線。

## 6. 重啟時建議的下一步（擇一）

1. **路 B（最小改動、最可能見效）**：把 `LDIBuilder` 注入的 inpainter 由 C1 換成
   **預訓練 LaMa / 輕量 diffusion inpainting**（選用安裝、import 失敗退 C1）。大洞填得可信後，
   破洞分支才會帶來肉眼可見的提升。**這是投報率最高的一步。**
2. **路 A’（純 CPU 改良補繪）**：在 C1 之上加 patch-based（PatchMatch 風格）或邊緣導向合成，
   減輕超大洞的放射狀 smear；不需重依賴，但天花板仍受古典法限制。
3. **路 C（換代）**：待 8GB 友善、單圖、可商用的 3DGS/feed-forward 方案成熟再回試（見 3DGS spike 結論）。

## 7. 給「尋找其他解法」的判準（避免再撞同一面牆）

- 任何宣稱「shader 就能補洞」的方案，受限於第 4.3 節天花板，**對大洞無效**；先確認它怎麼處理
  被遮擋內容（是真生成、還是又一個擴散近似）。
- 評估補繪器時，**直接看它對「整張床」這種超大洞的輸出**，不要只看小破洞 demo。
- 維持本專案的硬約束：**重依賴一律選用安裝 + import 失敗優雅退回**，不污染基礎部署。
